cityio ver3 Change Log

version 3 has less lines of code and has more features stepping towards a more version controlled archive than a simple json dump.

1 Breaking Changes

1.1 Stricter Protocol

1.1.1 No trailing slashes

curl http://locahost:8080/api/tables/list/

won’t work. traling slashes are not allowed. Very rare, but we had one time with multiple slashes that made cityio wonder the meaning of life. 

api/table/update/new_table////what-is-this////waht//doge-coin//

this was undefined. I suggest to ban this all together

1.1.2 Header will require ’Content-Type:application/json’

This caused different hashes due to encoding of raw bytes on body, I think the increase of overhead is acceptable if it’s used to properly indicate that this is json data.

Plus, I propose a method to compress bytes for heavy tables to save bandwith. brix might decode this.

1.2 Update endpoint

The method POST implies you are updating a table, from now we can use a shorter endpoint. 

curl -X POST -H "Content-Type:application/json" -d '{"new":"data"}' localhost:8080/api/table/new_table

1.3 Tables List endpoint

we had the full url output using the tables list update, https://cityio.media.mit.edu/api/table was hardcoded, this is wrong.

1.4 base58 encoding of Hashes

The hashing algrithms are base58 encoded now.

2 New things

2.1 rollback

tables saved in a linked list (forward) chain manner, you can roll back to previous versions of that table under the same name.

2.2 Deep Post

cityio does not limit you by module level posting, but till the end, if that object exists. Think of mkdir command but without the -p (–parent) option. This will update the module and table hash accordingly.

2.3 module level backup

Modules can be individually posted, you can ask cityio if this was previously calculated, and avoid doing it again + reduce the payload.

3 Upcomming

3.1 websocket server

enables miro like cursor movement for tables, we can think of collectively editing. What do you think Ariel?

3.2 Better yet simpler auth, cityio sends you e-mails!

user auth with no passwords limiting personal information, emails will not be saved in plain text, the server, yasushi will not be able to retrieve your address.
