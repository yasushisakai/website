8 Rollback (1/2)

Tables are saved in a linked list data structure, you can roll back to previous versions of the same table by using the id. 

curl https://cityiotest.mirage.city/api/table/test/

{
 ...
  "meta": {
    "hash": "8tFUU1YRaENHRBdsWg5q8FFC7jgwZi2Uxi9HAApDsVo1",
    "hashes": { ... },
    "id": "14Z4435BHHknD2F3jd68diL5NhMcNHspX7CcRTXVWGrT",
    "timestamp": "2021-03-27T21:15:44.869320546+00:00"
  },
 ...
}

The hash field is generated by concatenating the hashes. This is for comparing the uniqueness of the data. The id is to point a specific point within the history.

9 Rollback (2/2)

The GET commit endpoint will show information about the parent commit id. 

curl https://cityiotest.mirage.city/api/commit/14Z443...VWGrT/

{
  "tree_id": "8tFUU1YRaENHRBdsWg5q8FFC7jgwZi2Uxi9HAApDsVo1",
  "parent": "GmJxu8Lq4vT2xx2HaW9nrmTJdprdKkkrFYFkQ8Mq8UcC",
  "timestamp": "2021-03-27T21:15:44.869320546+00:00"
}

The parent field is another commit id so, we can get this again. 

curl https://cityiotest.mirage.city/api/commit/GmJxu8...Mq8UcC/

You can go on until the parent becomes "0" (origin).

10 Branching

Once we can jump back using rollback, similar to git tag you can manifest a previous version to show up as the HEAD of a table.

\vspace{10pt} 

curl -X POST \
https://cityiotest.mirage.city/api/table/raw/branch/GmJxu8.../

This will now add a table called branch. 

{"id":"branch","status":"ok"}

This data structure is better in saving history, and duplication cost is very small (only the pointers.)

11 Deep Post

You can now deep post (currently lets you deep get), if the path of that object exists. This is similar to mkdir without the -p or --parent option which recursively retrieves the object.

\vspace{10pt} 

curl -X POST \
-H "Content-Type:application/json" \
-d '{"cars":true, "bikes":false, "pev": true, "walk": false}' \
https://cityiotest.mirage.city/api/table/new_table/new_module/setting

{"commit":"GjM9wDpQJV5H5cGfZWnSKnJwtCimVQJB38RerPg2mfdm","status":"ok"}

12 Module Level Caching

Modules can be individually posted, and therefore will be cached. You can ask cityio if this was previously calculated, and avoid expensive calculation or sending redundant data. 

curl -X POST \
-H "Content-Type:application/json" \
-d '{"data":"blob"}' \
https://cityiotest.mirage.city/api/module/

This will return it’s module id. 

{"id":"3mFuwA1tfMJPRLxhqTcmRuZ43Yap98ohxgZmrwuNjNmR","status":"ok"}

curl https://cityiotest.mirage.city/api/module/3mFuwA...jNmR/

{"data":"blob"}
