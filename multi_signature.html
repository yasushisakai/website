multi-signature

trying to check if I understand the concept of it

the combination is \(nC_k\) where \(n\) is the total number of members, and \(k\) is the least required number of agreements.

\[ \ nC_k = \frac{n!}{k!(n-k)!} \]

for example, if there were three people (the usual three), how many keys should that multi-sig hold?

This will be \(3C_2\) which is 3.

the combination is:

alice-bob, alice-charlie and bob-charlie.

if all three agreed, that should still pass, but it’s effectively same as alice-bob 

import hashlib

alice = hashlib.sha256(b"alice").hexdigest()
bob = hashlib.sha256(b"bob").hexdigest()
charlie = hashlib.sha256(b"charlie").hexdigest()

# within the three people, it passes when more than two of the people agrees.


def msig(components):
    sig = []

    for i in range(len(components)):
        for j in range(i, len(components)):
            if (i == j):
                continue
            sig.push(hashlib.sha256(sorted([components[i],components[j]]).join().bytes('utf-8')).hexdigest())

    return sig

components = [alice, bob, charlie]

sig = msig(components)
